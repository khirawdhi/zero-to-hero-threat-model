# TM-SC-04 — Artifact Repository Threat Model

This threat model focuses on **security risks associated with artifact repositories**, which store build outputs such as binaries, packages, and container images before they are deployed to runtime environments.

Artifact repositories act as a **central distribution point** within the software supply chain. Once artifacts are stored and trusted within the repository, downstream systems—including deployment pipelines and production environments—often assume those artifacts are safe.

Because of this trust, a compromise of the artifact repository can enable attackers to **distribute malicious software to multiple systems simultaneously**.

---

## Scope

* Storage of build artifacts generated by CI/CD pipelines
* Artifact publishing workflows from build systems
* Artifact retrieval by deployment pipelines
* Access control and authentication mechanisms for artifact repositories
* Artifact integrity verification mechanisms

Typical repository platforms include:

* Nexus Repository
* JFrog Artifactory
* GitHub Packages
* AWS CodeArtifact

---

## Out of scope

* Security of the CI/CD pipeline generating artifacts (covered in **TM-SC-01**)
* Dependency supply chain risks from external registries (covered in **TM-SC-02**)
* Container build pipeline security (covered in **TM-SC-03**)

---

## Key assets

* build artifacts (binaries, packages, container images)
* artifact metadata
* artifact signing keys
* repository access credentials
* CI/CD service accounts publishing artifacts

---

## Typical Artifact Repository Flow

A simplified artifact storage and distribution pipeline:

```
CI/CD Pipeline
   ↓
Build System
   ↓
Artifact Repository
   ↓
Deployment Pipeline
   ↓
Runtime Environment
```

Artifact repositories may store multiple artifact types including:

* container images
* application binaries
* package artifacts
* dependency packages

---

## Security Objectives

The artifact repository should ensure:

* integrity of stored artifacts
* authentication and authorization for artifact publishing
* traceability of artifact creation and distribution
* protection of signing keys and repository credentials
