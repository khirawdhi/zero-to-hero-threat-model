# Data Flow Diagram Guide

Data Flow Diagrams (DFDs) are used to visualize system architecture and identify trust boundaries.

## Key Elements

| Element | Description |
|---|---|
| Process | system component performing operations |
| Data Store | storage system |
| External Entity | user or external service |
| Data Flow | communication between components |
| Trust Boundary | change in security context |

## Best Practices

- keep diagrams simple
- highlight trust boundaries
- include external systems
- identify sensitive data flows
