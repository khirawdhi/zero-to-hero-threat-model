# Threat Modeling Lifecycle

Threat modeling should be integrated into the **system design lifecycle**, not treated as a one-time security activity.

## Typical Lifecycle

1. Architecture Design
2. Threat Modeling Session
3. Risk Identification
4. Mitigation Planning
5. Implementation
6. Security Validation
7. Continuous Review

## Integration into Engineering

Threat modeling is most effective when performed during:

- architecture design reviews
- major system changes
- introduction of new dependencies
- infrastructure migrations
